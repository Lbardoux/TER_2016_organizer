Agenda.Agenda --+

               Dependance

La classe qui définit une dépendance pour un Agenda. Actuellement, une dépendance est un Agenda, dont les créneaux sont considérés comme des Blocages, et qui est immuable (comprendre que, en tant que dépendance, on ne peut le modifier directement). Attention, on ne doit pas s'en servir comme Agenda principal (ie son attribut pere ne doit jamais etre None !)

ajouterCreneau(self, annee, mois, jour, debut, fin, typeCreneau=CP.CRENEAU)

source code 

Surcharge de cette méthode pour prendre en compte les spécifications d'une Dependance. Cela creera forcément un blocage Etape 1 de la descente dans l'architecture. Ceci va "ajouter" un Creneau dans le mois de l'annee annee, jour, entre debut et fin. Noter que si annee n'existe pas, elle sera automatiquement créée.

Parameters:
  • self - L'argument implicite
  • annee (int) - l'annee dans laquelle insérer ce créneau.
  • mois (int) - le numéro du mois dans lequel insérer ce créneau.
  • jour (int) - le numéro du jour dans lequel insérer ce créneau.
  • debut (int [1, 48]) - l'heure de début du créneau
  • fin (int [1, 48]) - l'heure de fin du créneau
  • typeCreneau (enum) - une valeur enumérée pour la fabrique de creneau
Returns: Creneau
Un créneau nouvellement créé.
Raises:
  • ValueError - si les arguments sont erronés.
Overrides: Agenda.Agenda.ajouterCreneau

Precondition: debut < fin, mois/jour/debut/fin doivent etre compris dans leurs intervalles respectifs
